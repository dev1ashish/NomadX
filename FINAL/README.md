# Atlas Raman Classifier — FINAL Deliverable

Four-class confocal Raman classifier (STEC / Non-STEC E. coli / Salmonella / H2O)
trained on 87 spectral files (7,122 QC-passed pixels on a 987-bin axis).
The project headline is **PLS-DA LOSO 0.603 macro-F1 on raw spectra**, with a
parallel engineered-feature track reaching **LogReg-L2 LOSO 0.448 file-weighted
macro-F1** (95% CI [0.345, 0.552]) on 35 mutual-information-selected features.
The engineered-feature classifier is significantly better than PLS-DA on the
same feature space (McNemar p=0.0020), and the strongest single STEC↔Non-STEC
discriminator is `mcr_C6_mean` (Cohen's d = -1.23, global-fit).

## How to reproduce

The notebooks expect to run from the **project root** (one level above this
`FINAL/` folder), alongside the existing `data_cache/`, `artifacts/`, and
`outputs/` directories produced by the upstream pipeline. The recommended
workflow is:

```bash
# from the project root
python -m venv .venv && source .venv/bin/activate
pip install -r FINAL/requirements.txt
pip install jupyterlab   # not in requirements.txt (which is the slim deploy set)
jupyter lab FINAL/notebooks/
```

Run the notebooks in numeric order. Notebooks 01-02 build / verify the data
cache; 03 reproduces the PLS-DA headline; 04-07 produce the engineered-feature
caches; 08 runs the mixed-sample stress test; 09-10 fit the final classifier
and compute statistical tests; 11 demonstrates inference on a fresh `.xls`
file using the saved Stage 15F artifact.

The dev environment also needs `torch`, `jupyter`, `umap-learn`, and
`seaborn`; these are intentionally **not** in `requirements.txt`, which is
the slim inference + notebook set.
